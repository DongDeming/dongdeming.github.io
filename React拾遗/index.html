ref

React 中出现ref 的方式有

  • React.forwardRef()：ref 转发
    • const FancyButton = React.forwardRef((props, ref) => (...)) 允许组件接收 ref prop，并向下转发给子组件
    • 使用：<FancyButton ref={ref}>
    • 仅能用于函数组件，类组件要转发ref 可以另起名称，如this.props.innerRef
  • React.createRef() 创建一个 ref
    • 可以作为组件的ref prop, 其.current 属性指向该组件实例
    • 可以作为原生DOM的 ref prop, 其.current 属性指向该DOM元素
  • useRef() 创建一个贯穿函数组件生命周期的 ref实例
    • 可以修改该实例的.current值达到缓存数据的目的，.current 的变化不会触发组件的重新渲染
    • 可以用于组件的 ref prop
  • 组件ref传入回调函数，ref 作为函数参数：函数中接受 React 组件实例或 HTML DOM 元素作为参数，以使它们能在其他地方被存储和访问

代码分割 React.lazy()

  • 核心是使用动态引入const OtherComponent = React.lazy(() => import('./OtherComponent'))
    • webpack 能够识别import() 并完成代码分割
  • 结合<Suspense> 组件能够实现组件加载的过程控制，如显示Loading页面作为替代
  • 组件加载失败会报错，所以应该使用 Error Boundaries 组件包裹处理错误

Error Boundaries组件

目的是为了捕获子组件渲染期间 render方法、生命周期方法、构造函数 的 JavaScript 错误

  • 实现了 static getDerivedStateFromError() or componentDidCatch() 声明周期函数的组件

  • 捕获错误之后，能够渲染出备用 UI

  • 无法处理以下场景的错误

    • 事件处理

    • 异步代码（例如 setTimeout）

    • 服务端渲染

    • 它自身抛出来的错误

  • 其它处理错误方案

    • try/catch， 但无法捕获异步报错
    • promise catch 处理promise报错
    • 可以用 unhandledrejection 捕获全局的异步报错
    • window.onerror 处理全局非异步报错
    • unhandledrejection 和 onerror 都是全局事件，正常情况下不能用来写业务逻辑

Portal

  • Portal 提供了一种将子节点渲染到存在于父组件以外的 DOM 节点的优秀的方案
  • 但事件冒泡机制（合成事件）还是走父子组件这种关系的事件流，而非原生真实的DOM节点关系
  • 适用对话框、悬浮卡以及提示框等

DOM Elements

  • 属性名必须用小写的驼峰命名
  • data-=*， aria-* 仍保留小写
  • dangerouslySetInnerHTML: 替代innerHTML的写法
  • react 支持所有的 HTML 属性
