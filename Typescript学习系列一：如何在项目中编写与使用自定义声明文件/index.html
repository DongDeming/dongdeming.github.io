Typescript学习系列一：如何在项目中编写与使用自定义声明文件

发表于 2020-07-04 | 分类于 Typescript |

一、什么是声明文件（*.d.ts）
声明文件就是针对已经存在的变量，函数，类等声明其数据结构，调用结构，以便在使用的时候能产生智能提示，语法检查。从而帮助我们更好的编写代码

二、项目中如何编写自定义的声明文件并使用？

  1. 全局脚本文件（非模块，不能以模块的方式引入）global.d.ts
    • 文件中不含 import export等模块相关的导入导出语句
    • 可以放在项目的任何位置，然后通过如下方式引入
      • 在tsconfig.json文件的files includes 中包含该文件路径
      • 在使用的文件中通过注释引入：/// <reference path="global.d.ts" />
  2. 私有模块
    • 文件中含import export等模块相关的导入导出语句
    • 需要通过模块引用的方式引入，如：import {typeA} from 'path'

Ps. --isolatedModules: boolean; // tsconfig.json配置字段：要求将 *.ts 文件作为一个单独的模块，因此必须含 import 或 export 。但并不针对 *.d.ts
