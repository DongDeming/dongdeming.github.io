typescript基础一

发表于 2020-11-02 | 分类于 Typescript |

TypeScript 最大的优势之一便是增强了编辑器和 IDE 的功能，包括代码补全、接口提示、跳转到定义、重构等。

TypeScript 是 JavaScript 的一个超集，主要提供了类型系统和对 ES6 的支持

TypeScript 核心库的定义文件 内置对象是指根据标准在全局作用域（Global）上存在的对象。这里的标准是指 ECMAScript 和其他环境（比如 DOM）的标准。TypeScript 核心库的定义中不包含 Node.js 部分，需引入npm install @types/node --save-dev。

JS 类型

  • 基础值类型：字符串 String 、布尔 Boolean、数值 Number、对空 Null、未定义 Undefined、Symbol
  • 引用数据类型：对象 object （数组 Array、函数Function 等）

对应的TS 类型

基础类型 说明
string ‘, “, 模版字符串
boolean true, false
number 支持2, 8, 10, 16进制字面量
null、undefined 默认情况下： null, undefined 是所有类型的子类型，即可以赋值给其他任何类型(never除外)
当指定了 --striceNullChecks 则只能赋值给他们各自和其父类型（any, unknown, undefined 的父类型是void）
Symbol ES6 引入了一种新的原始数据类型，表示独一无二的值。var s: symbol = Symbol('hello');
object object表示非原始类型
数组： let x: number[]; let x: Array<number>;

TS扩展类型

扩展类型 说明
元组 数组长度，每个位置的类型已知 let x: [number, string]
当访问一个越界的元素x[5] = 'hello'; // true x[6] = true; // error，会使用联合类型替代：即现有元组中元素类型的联合类型 number ｜ string
enum 与其他语言的枚举类似，默认情况下，从0开始为元素编号，可以指定；
Color.Red 得到的是数字， Color[1] 得到的是字符串
any 使用它将跳过类型检查以及类型推导。适合用于javascript迁移到ts，
适用于标记那些可能来自于动态的内容(编程阶段还不清楚类型的变量),比如来自用户输入或第三方代码库。
void 表示没有任何类型。
当一个函数能执行到结束，没有返回值时，你通常会见到其返回值类型是 void
通常应用于函数返回值
never 函数 无法执行到结束: 如报错，死循环；（通常应用于函数返回值）,
never 类型仅能被赋值给另外一个 never; 即使 any也不可以赋值给never。
never类型是任何类型的子类型，也可以赋值给任何类型
通常应用于函数返回值
unknown unknown: 推荐使用 unknown 替代 any。来标记哪些可能来自于动态的内容。
任何值都能赋给unknown变量，同any
unknown 变量必须转换成其他变量才能使用属性操作符

类型断言

TypeScript会假设你，程序员，已经进行了必须的检查。将跳过类型检查。但类型断言还是有限制的，将在其他篇幅详解。

其他

  • tsc ts 转换成js的cli命令
    • ts中的类型注释是轻量级的，仅做静态代码分析，而非运行时校验；在执行转换的过程中，尽管校验上不符合提示，但最终的js代码还是会生成
  • 在TypeScript里，只在两个类型内部的结构兼容那么这两个类型就是兼容的。（鸭子？）
  • 类：constructor(public firstName, public middleInitial,public lastName); 使用public等同于创建了同名的成员变量。

常用配置

参数 说明
noEmitOnError 抱错时终止js文件的生成
outDir 编译结果输出
declaration 将由.ts文件自动生成 .d.ts文件
declarationDir 设置生成 .d.ts的目录
declarationMap 生成 .d.ts.map文件
emitDeclarationOnly 仅生成 .d.ts 不生成js文件
