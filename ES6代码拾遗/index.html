async/await

-  `async function`声明函数为异步函数，并保证函数的返回值为 `promise`实例
- `await` 只能在 `async function`中使用，用于等待一个`promise` 对象的结构，会同步阻塞后续语句的执行。

Class 实例属性的初始值由调用实例方法生成

相当于

  • 这种写法相当于在constructor 执行的最后插入该实例方法进行赋值，调用顺序为实例属性的声明调用顺序

  • 这种写法容易搞错调用顺序，没有明显的触发调用痕迹，不建议使用

promise 处理完成之后继续调用then的方法

- 没有处理之前注册的`then, error`中的回调方法都会等到状态变化之后触发。
- 处理之后注册的`then，error`中的回调方法则会立即异步被异步调用，拿到同样的处理结果。

装饰器

装饰器的本质是一个函数，只能作用类、类静态、动态成员。语义结构清晰，能够在不破坏类内聚的基础上，给类提供额外功能特性。装饰器在编译期会被转译为相应的表达式语句。

使用前准备

由于装饰器在目前还是一个提案，因此使用使用装饰器语法的时候 ，需要设置babel或typescript的相关配置。

  • babel 需要安装语法支持插件 [babel-plugin-syntax-decorators](https://www.npmjs.com/package/babel-plugin-syntax-decorators)

  • typescript 需要配置tsconfig.json

第三方库

core-decorators.js，提供了几个常见的装饰器，部分如下：

- **@autobind**: `this`绑定
-  **@readonly**：属性只读
- **@override**：覆盖原方法
- **@deprecate** (**@deprecated**)：废弃标识

this 绑定解析

绑定优先级从上到下降低

  • 构造函数｜箭头函数

  • 显式绑定( call apply bind )

    • 传入的绑定对象是非对象的话走 默认绑定 逻辑

  • 隐式绑定(obj.fn())

  • 默认绑定（globalThis，严格模式下是undefined）

严格模式

严格模式下 默认绑定 的this将是undefined

使用场景

  • 全局环境： this === globalThis
  • Node 文件模块： this === module.exports
  • 构造函数： this 为当前new创建的实例
  • 函数：this 为 默认绑定 结果
  • 对象方法：this 为 隐式绑定 结果
  • 箭头函数：this为当前词法上下文
    • class 中方法的箭头函数相当于在构造函数中的this.a = () => {} 怎样的声明，this指向当前实例，而非原型链
    • 如果词法环境的 this是动态的，箭头函数的this也会随着变化：如函数中声明箭头函数，此时箭头函数的this用于指向函数中的this，但函数中的this可以随调用方式而变化。

Let\const核心概念

  • 块作用域

  • 声明不再提升

  • 暂时性死区：块作用域内必须先声明再使用，否则报错

  • for循环还有一个特别之处，就是设置循环变量的那部分是一个父作用域，而循环体内部是一个单独的子作用域。（虽然在循环体内设置i能影响父作用域的i（如：i = 2；会影响循环次数）。但每一次循环，循环体内的i都相当于是一个新的变量，即let i = i，因而产生的闭包不受父作用域中i的影响）

  • 应该避免在块级作用域内声明函数。如果确实需要，也应该写成函数表达式，而不是函数声明语句（因为有歧义）。

解构：

  • 解构可以有默认值，当严格等于undefined时取默认值
  • 非声明式/参数式的解构应该放在括号中，({x} = {x: 1})，否则{x}可能会被看成块作用域
  • 数组：解构的值不存在则是undefined, 右侧被解构的值不是iterator对象则报错
  • 对象：解构的值不存在则是undefined, 右侧如果是null undefined 则报错，原生值会被封装成对象。
  • 解构不一定会创建新变量，也可能是对原有变量赋值，如[a, b] = [b, a];

字符串：

  • ”\uxxxx” 可以表示码点在\u0000~\uFFFF的字符。超出的需要使用双字节。如："\uD842\uDFB7"表示 ”𠮷”。新表示法：”\u{20BB7}”
  • for...of，解构[…str]能识别双字节码点，for循环无法识别
  • 模板字符串默认会将字符串转义，导致无法嵌入其他语言。
  • 处理字符串，或JSON操作时，如果含有不可见的字符，以及转译符，要谨慎查阅文档之后处理。
  • 要时时刻刻考虑双字节字符的存在。

函数

  • 尾调用（某个函数的最后一步是调用另一个函数），尾递归（某个函数的最后一步是调用自身）
  • 函数调用会在内存中形成调用记录（”调用帧”），用于保存调用位置和内部变量等信息。尾调用由于是函数操作最后一步，不需要保留外层“调用帧”。这将大大节省内存，这就是“尾调用优化”的意义。
