homebrew安装记录

发表于 2020-12-31 | 分类于 实践案例 |

  1. 开启代理，没有代理请参考homebrew官网查看其它安装方式

  2. 安装 home-brew : /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

    1. 碰到问题：curl: (7) Failed to connect to raw.githubusercontent.com port 443: Connection refused

    2. 打开 https://www.ipaddress.com/ 输入访问不了的域名raw.githubusercontent.com 查询到 ip:199.232.68.133

    3. 在hosts 中添加代理

    4. 重新安装，期间会有几次加载到一半失败的情况。重复运行安装命令即可

  3. brew 软件安装查找网址

  4. 解决brew install... 过慢

    1. 问题定位: 使用brew update --verbose观察 update 过程：

      发现 update 卡在从 github 仓库获取文件的过程。这个结果与手动从 github 下载文件慢的症状相互印证。

    2. 解决

      由于问题主要是在国内网络环境 github 下载慢，因此尝试：

      1. 更换使用国内的 homebrew 镜像源；
      2. 使用代理访问 github.com。

    3. 更换 Homebrew 源

      使用以下命令更换国内阿里云上的 homebrew 镜像：

      替换后，问题依旧，继续查看日志：

      可以看到由于homebrew-cask的仓库依然指向了 Github，这个过程还是慢。阿里云的 镜像站 没有提供homebrew-cask，进一步搜索找到 USTC 镜像站，该站提供了homebrew-cask的源。使用上述同样的命令更换源：

      测试发现问题解决。

    4. 使用代理

      家里的路由器已经配置好了 SS 代理，只要简单把github.com加入到黑名单强制走 proxy 就可以了。然而代理速度一般，效果不及上述方法。同时为了保证在其他网络环境下的效率，保留方法一所做的修改。

    5. 技巧

      1. 查看安装详细过程 brew install --cask xmind --verbose

      2. 由于加载缓慢，经常会出现失败的情况

      3. 观察日志可以发现

        1. 下载地址为：https://dl3.xmind.net/xmind-8-update9-macosx.dmg
        2. 本地目标地址与名称为：/Users/yourusername/Library/Caches/Homebrew/downloads/4619cef91fde2f32eb11308c15300d68fb0e7e73b569ca799c4f2d44d02a27cb--xmind-8-update9-macosx.dmg.incomplete

      4. 在浏览器中下载软件：https://dl3.xmind.net/xmind-8-update9-macosx.dmg

      5. 修改软件名称并保存为/Users/yourusername/Library/Caches/Homebrew/downloads/4619cef91fde2f32eb11308c15300d68fb0e7e73b569ca799c4f2d44d02a27cb--xmind-8-update9-macosx.dmg

      6. 再次执行 brew install --cask xmind --verbose

      7. 成功安装 xmind

    6. mac pro M1 oh-my-zsh安装教程

  5. vscode命令行执行brew install lerna 报错：

    1. Error: Cannot install in Homebrew under Rosetta 2 in ARM default prefix (/opt/homebrew)!
    2. 解决：在 iterm2 下执行brew install lerna
