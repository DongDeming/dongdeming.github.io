typescript基础二：接口

发表于 2020-11-05 | 分类于 Javascript |

基础

接口与对象

接口可以用来声明一个对象的属性及其类型

  1. 对象类型检查使用的是鸭子辨型：即只关注属性是否存在，以及对应的属性类型是否匹配。
  2. 例外 额外属性检查：当对象字面量被赋值给变量或作为实参传递的时候，会触发 *额外属性检查。即当字面量对象存在任何 *目标类型 不包含的属性时，就会导致类型检查错误，规避方案：
    • 目标类型 包含所有属性：如使用 索引签名 [propName: string]: any 包含任何属性
    • 先将字面量赋值给变量，然后传递变量给 目标类型 ，此时不会触发 额外属性检查
  3. TS 支持两种索引签名：字符串和数字 。一个对象中可以同时使用两种类型的索引，但是数字索引 的类型必须是 字符串索引类型的子类型。
    • 因为使用 number 索引时，JS 会将它转成 string 再去索引对象。即 [100]等同于 ['100']，两者需要保持一致
    • 字符串索引会强制要求其他属性与它兼容，否则会抱错

接口与接口

  1. 一个接口可以继承多个接口 interface Square extends Shape, PenStroke {} 。当然要注意多个被继承的接口的兼容性

  2. 接口只能扩展具有可选参数的标识符/限定名称

类继承接口 implements

  1. 类是具有两种类型的：静态部分的类型和实例部分的类型
  2. 当一个类实现一个接口时，只对其实例部分进行类型检查
  3. coustructor 函数属于类的静态部分，因此不在接口的类型检查范围内。（可以通过函数声明构造器约束）
  4. 接口描述了类的公共部分，而不是公共和私有两部分。 它不会帮你检查类是否具有某些私有成员。

接口继承类

  1. 接口继承类的实例部分的成员但不包括实现
  2. 接口同样会继承到类的private和protected实例成员。
    • 这意味着当你创建了一个接口继承了一个拥有私有或受保护的成员的类时，这个接口类型只能被这个类或其子类所实现
    • 即私有和保护属性必须同源

混合类型

一个对象可以同时做为函数和对象使用，并带有额外的属性
