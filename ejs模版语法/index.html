Ejs： 嵌入式javascript模版

ejs 是一款非常简单易用的 javascript 模版库，使用 ejs 能够帮助我们快速的构建动态的 html 页面。其原理是将 template 根据 options 编译成（compile）成函数( 后面简称模版函数)。 然后根据数据 data 生成字符串内容。

特性

- 快速编译与绘制输出（支持调试）
- 简洁的模板标签： `<% %>`
- 自定义分割符（例如：用 `[? ?] `替换` <% %>`)
- 支持模版片段
- 支持客户端/服务端
- 支持 `js` 中间结果的缓存
- 模版静态缓存 🤔️？
- 兼容 `express` 视图系统

简洁的模版标签

前缀标签 解释
<% ‘脚本’ 标签，用于流程控制，无输出
<%= 输出数据到模板（输出是转义 HTML 标签）
<%- 输出非转义的数据到模板，常用于接受 include 模版片段的结果
<%_ 删除其前面的连续多个空格符（而非空白），用于流程控制，无输出
<%% 输出字符串 ‘<%’, 无需后缀
<%# 注释标签，不执行、不输出内容
后缀标签 解释
%> 正常结尾
_%> 将结束标签后面的连续空格符删除（而非空白），测试发现如果连续空格后是换行，则将换行符也删除，然后停止
-%> 删除紧随其后的换行符（而非所有空白）

Options 字段解析

字段 解释 例子
cache 是否启用缓存缓存 ejs.cache._data["filename"] 中将会出现缓存数据，同时ejs.cache提供了 get/set功能
filename 缓存字段名 ejs.cache._data["filename"]
context 指定模版中的this绑定的上下文 <%= this.name %>
compileDebug 是否需要在模版函数中输出调试语句 <% debugger; %>
client 是否生成一个能够任何地方直接使用，不再依赖其他包的 模版函数 模版函数(data)能在任何地方执行，并得到同样的字符串内容
delimiter 放在角括号中的字符，用于标记标签的开与闭 如设置为 ? 则可以用 <? ?>声明模版
debug 在命令行窗口中打印模版函数代码
_with 模版函数中是否使用 with() {} 获取局部变量 <%= people.name %> <%# _with 默认为 true %><br/><br /><%= locals.people.name %> <%# _with 设置为 false %>
localsName _with=false,然后自定义locals 对象名称**
rmWhitespace 删除模版中所有可安全删除的空白字符，包括开始与结尾处的空格。比<%_, _%>, -%>更加激进
escape 为 <%= 结构设置对应的转义（escape）函数
outputFunctionName compile 中會把 outputFunctionName 取出來拼接進模板，控制这个变量能写任意的逻辑代码 {outputFunctionName: 'a = 12; var sayHi',} // var [outputFunctionName] = __append; => var a = 12; var sayHi = __append;
async 当值为 true 时，EJS 将使用异步函数进行渲染。（依赖于 JS 运行环境对 async/await 是否支持），适用于模版中存在异步函数的调用 <% let data = await someFunction()%}
