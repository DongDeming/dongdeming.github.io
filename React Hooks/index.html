React Hooks

发表于 2021-03-24 | 分类于 React |

Hooks是什么

hooks 是 react16.8.0 推出的解决 函数组件 没有状态、没有生命周期的一套机制。使得 函数组件 能够替代 类组件 以及在某种程度上超越 类组件

Hooks 解决了如下问题

  1. 为函数组件提供了状态与生命周期
  2. 通过自定义hooks 扁平化的实现状态逻辑的复用。而 render props ，HOC 会带来层次嵌套的冗余，复用的逻辑越多，嵌套越深。
  3. 副作用的关注点分离, 使用专门的 useEffect useLayoutEffect 来处理副作用相关的操作
  4. 细分状态，可以设置多个state，并单独针对这些state操作，实现状态以及相关操作的解耦
  5. 细分不同声明周期函数中的逻辑, 类组件中同一个声明周期函数中放置了太多不相关的逻辑
  6. Hooks 要集中在 UI 函数顶部写，也很容易养成书写无状态 UI 组件的好习惯，践行 “状态与 UI 分开” 这个理念

编写规则

  1. 只能在函数组件的顶层以及自定义 hook 的顶层词法环境使用，不能被其它词法环境嵌套(如条件语句)。这样是为了确保 hook执行的顺序
  2. Hook 函数必须以 use 命名开头
  3. 编写语法检测插件：eslint-plugin-react-hooks

基础 hooks

Object.is():hooks 通过Object.is() 比较差异判断是否需要进行更新

  • 除了both +0 both -0 both NaN 限制外，基本跟 === 全等相同

useState

  • const [count, setCount] = useState(0)
    • 只执行一次初始化
    • setCount 不管重复渲染多少次都是同一个函数（引用相同）
    • count 永远全等于 setCount()设置的值，如果 count是对象，但组件被其它状态的变更触发 rerender 时 count 不变（引用相同）
    • setCount() 通过 Object.is() 判断是否触发更新
    • 状态的变化能够触发组件的重新渲染

useEffect

  • 异步，DOM渲染之后运行，类似于 compoenntDidMount, componentDidUpdate 生命周期函数
  • 返回值是函数的话，返回值类似于 componentWillUnmount 生命周期函数
  • 没有依赖数据则每次render 都执行，数组为空则仅执行一次，数组非空则根据Object.is()算法判断是否要再次执行
  • 用于副作用处理，如接口调用，DOM节点查找等与外部对接

useContext

  • 需要与 const MyContext = React.createContext(defaultValue) 配合使用
  • useContext(MyContext)：当组件没有<MyContext.Provider> 包裹时取到 defaultValue，否则取到<MyContext.Provider> 的 value prop
  • 取代MyContext.Consummer, 减少了层次嵌套的冗余
  • 即使用了 React.memo, 只要Context值（Object.is()判断）发生了变化，仍让会触发重新渲染
  • MyContext.Provider 设置的 value 与 useContext 的取到的值是全等的，value 不变的情况下多次rerender 取得的值时相同的
  • 提供了一种非props, 跨组件或突破组件间界限的存取值/状态的方案，适用 global数据
  • 使用useContext使得数据并不一定从props传入，会使得组件变得难以复用，要谨慎使用

额外的 hooks

useReducer

const [store, dispatch] = useReducer(reducer, init)

  • 只执行一次初始化
  • dispatch 不管重复渲染多少次都是同一个函数（引用相同）
  • store 永远全等于 reducer设置的值，如果 store是对象，但组件被其它状态的变更触发 rerender 时 store 不变（引用相同）
  • reducer()执行生成的store 通过 Object.is() 判断是否触发更新
  • store的变化能够触发组件的重新渲染

useCallback

  • 返回一个被记忆的函数，依赖数组的值不变，则多次rerender都反回同一个函数
  • 与React.memo的子组件配合，确保函数传递给子组件的是同一个引用，减少不必要的渲染

useMemo

  • 返回一个被记忆的执行结果，依赖数组的值不变，则多次rerender都不会再执行函数，而是返回同一个结果
  • 与React.memo的子组件配合，确保值传递给子组件的是同一个引用，减少不必要的渲染

useRef

  • 创建一个整个声明周期都不变的对象, 仅初始化时候执行一次
  • 可修改的属性是 .current，修改 .current 不会导致组件的重新渲染

useLayoutEffect

  • 与 useEffect 基本相同
  • 不同点：会阻塞DOM的渲染，useEffect 在全部渲染完毕后才会执行，useLayoutEffect 会在浏览器 layout 之后，painting 之前执行。
    • 这一阶段虚拟DOM已经映射到了真实DOM，但由于 JS 线程和浏览器渲染线程是互斥的，页面还没有渲染。componentDidMount，componentDidUpdate 以及 useLayoutEffect(create, deps) 的 create 函数都是在这个阶段被同步执行。
    • 建议将修改 DOM 的操作里放到 useLayoutEffect, 仅用一次回流、重绘的代价，就把所有需要更新的 DOM 节点全部更新完成。
    • 真实 DOM 已更新，即使没有渲染，useLayoutEffect(create, deps) 的 create 函数能拿到新的DOM
  • useLayoutEffect 相比 useEffect，通过同步执行状态更新可解决一些特性场景下的页面闪烁问题。

useDebugValue

  • 给自定义hook 打印相关信息用，这些信息会在DevTools中展示

自定义Hooks

  • 是一个函数，已use命名开头，并在函数体顶层词法环境调用其他hooks
  • 入参，返回值无强制要求，无需按照useState那样返回数组

生态圈扩展

  • Redux 提供了 Hooks API
  • React Router 提供了 Hooks API
