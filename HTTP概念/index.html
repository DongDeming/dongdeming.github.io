开发者手册/HTTP概念

发表于 2022-07-29 |
层次 协议
应用层 HTTP DNS
传输层 TCP
网络层 IP ARP
数据链路层

[URI][URI] 格式： http://user:pass@www.example.com:80/dir/index.html?uid=1#ch1

片段 含义
http 协议名称
User:pass 登陆信息
www.example.com 服务器地址
80 服务器端口号
dir/index.html 带层次的文件路径
uid=1 查询字符串
ch=1 片段标识符

HTTP规范了浏览器/服务器间数据传输的规范，但是在实际上有些客户端/服务器间并不遵守其规范（各大互联网公司的协议其实只要符合tcp/ip标准就能允许，目的有保证数据不被窃取，反网络爬虫等）

HTTP

  1. 服务器不会主动响应，除非客户端先发送请求

    请求报文
    方法
    URI
    协议版本
    可选的请求首部字段
    内容实体
    响应报文
    协议版本
    状态码
    原因短语
    可选的响应首部字段
    内容实体

  2. 是无状态协议（stateless），因此有了cookie技术

    请求方法 说明
    GET 获取资源
    POST 推送信息（常用于创建）
    PUT HTTP/1.1的PUT方法自身不带验证机制，任何人都可以上传文件（但可以通过WEB程序的验证机制验证，如REST架构）；常用于覆盖式的更新，并希望确保重复操作的结果相同（幂等性）
    HEAD 不返回报文主体，用于确认URI的有效性以及资源的更新时间等
    DELETE HTTP/1.1的DELETE方法自身不带验证机制，任何人都可以上传文件（但可以通过WEB程序的验证机制验证，如REST架构）；删除指定资源
    OPTIONS 查询服务器支持的请求方法
    CONNECT 要求与代理服务器通信时建立隧道（tunel），实现用隧道协议进行TCP通信，主要使用[SSL][SSL] 和 [TLS][TLS]协议加密内容。之后该服务器会代替客户端与目标主机建立连接。该代理服务器会向客户端发送或者接受TCP消息流；让服务器代替用户去访问其它网页，之后把数据原原本本的返回给用户。这样用户就可以访问到一些只有服务器上才能访问到的网站了，这就是HTTP代理。

  3. 持久化（HTTP keep-alive）

    1. 每次请求建立连接与断开，会增加通信量的开销（减少这部分开销）
    2. 持久化特点：只要任意一端没有明确提出断开连接， 则保持TCP连接状态
    3. HTTP/1.1 所有的连接默认都是持久连接

问题

  1. HTTP 每次都需要通过3次握手来确保数据已经准确送达了，为什么后续的持久化相关的图片中没有出现3次握手？（每个报文）
    1. A：图片中的是HTTP请求，而非3次握手。首次3次握手只是建立连接通道

扩展

  1. CGI(Common Gateway Interface) 通用网关接口
  2. REST(Representational State Transfer，表征状态转移) 架构
